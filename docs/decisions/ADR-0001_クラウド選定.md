# ADR-0001: デプロイ先にOracle Cloud Always Freeを採用

## ステータス
承認済み

## コンテキスト
既存のレンタルサーバ（Kagoya / CentOS7）は自社ホームページ運用中で停止不可、Pythonも利用不可。
社内セキュリティ規則により「外部→社内へのアクセス禁止」のため社内サーバへの設置も不可。

## 決定
Oracle Cloud Always Free（ARM 4CPU / 24GB）を採用する。

## 理由
- 無償で継続利用可能（約100名規模・軽量なアプリのため十分なスペック）
- HTTPS(443) outboundのみで社内ルールと整合（社内→クラウドへのアクセスのみで完結）
- 会社メールを持つ社員のみが利用対象のため、外部公開範囲を限定しやすい

## 代替案と却下理由
- 既存レンタルサーバ: Python非対応、CentOS7がEOL、既存ホームページと共存不可
- 社内サーバ: 「外部→社内アクセス禁止」の社内規則に抵触

## 影響・リスク
- Oracle Cloud Free枠にはSLA保証がない（`09_非機能要件.md` 参照）
- 無料枠の仕様変更リスクは継続的に監視する必要がある
