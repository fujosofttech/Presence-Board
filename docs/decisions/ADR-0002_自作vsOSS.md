# ADR-0002: OSS（GroupSession等）を採用せず自作する

## ステータス
承認済み

## コンテキスト
在席管理・行先共有を実現するOSSとしてGroupSession、Aipo、LibreBooking、Snipe-ITを検討した。

## 決定
既存OSSは採用せず、Django/PostgreSQLベースで自作する。

## 理由
- GroupSessionは最有力候補だったが、独自の運用ルール（8種の在席状態、行先必須/任意の状態別制御、事前登録フロー等）に合わせるには専用開発の方が柔軟
- Aipoは更新停止のため非推奨
- LibreBookingは会議室予約向けで用途が異なる
- Snipe-ITは資産管理ツールで対象外
- 開発チームがLinux/Django/PostgreSQL/Docker/Proxmoxに慣れており、自作の学習コストが低い
- 自作することで、AIコーディングエージェント活用・CI/CD導入の教育教材としても活用できる（`01_プロジェクト概要.md`）

## 代替案と却下理由
上記の通り、各OSSは機能要件または保守性の観点で不採用とした。

## 影響・リスク
- 自作のため、開発・保守工数はOSS導入に比べて大きい
- 教育目的と両立させるため、設計書（本ドキュメント群）を充実させることでリスクを軽減する
