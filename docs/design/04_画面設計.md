# 04_画面設計

> Version: 0.2
> 更新日: 2026-07-10

---

# 1. 文書の目的

本書は社内行先・在席管理システムの画面設計を定義する。

本システムは、現在社内で運用しているホワイトボードによる行先掲示板を電子化し、PCおよびスマートフォンから利用できるようにすることを目的とする。

画面設計では「ホワイトボードと同等以上の操作性」を最優先とする。

---

# 2. 設計方針

## 2.1 基本方針

- ホワイトボードに近い一覧表示を採用する
- 一覧画面から直接状態変更できる
- 画面遷移を最小限にする
- 社員の状況を一目で把握できることを重視する
- リアルタイム更新に対応する

---

## 2.2 PC版とスマートフォン版

本システムはレスポンシブ対応を行うが、利用シーンが異なるためPC版・スマートフォン版でレイアウトを最適化する。

|端末|目的|
|------|------|
|PC|社員全体の状況確認・状態変更|
|スマートフォン|自分の状態変更・社員状況確認|

## 表示順

表示順は以下の優先順位とする。

1. ログイン中の利用者（最上部固定）
2. 所属グループ
3. 氏名（五十音順）

---

## 一覧画面機能

- 所属課による絞り込み
- 氏名検索
- リアルタイム更新（SSE）
- 更新行のハイライト表示
- 一覧から直接状態変更

# 5. 状態変更ダイアログ

社員名クリックまたは状態クリックで表示する。

```
状態

▼ 客先

----------------------------

行先

_____________________

予定開始

2026/07/10 13:00

予定終了

2026/07/10 15:00

[保存]

[キャンセル]
```

入力項目は状態によって切り替える。

---

# 6. 状態ごとの入力項目

|状態|行先|予定期間|
|------|------|------|
|在席|-|-|
|客先|必須|必須|
|外出|必須|必須|
|会議|必須|必須|
|出張|必須|必須|
|客先常駐|-|-|
|在宅勤務|-|-|

# 7. 状態色

状態は色で判別できるようにする。

|状態|色|
|------|------|
|在席|緑|
|客先|水色|
|外出|オレンジ|
|会議|青|
|出張|紫|
|客先常駐|青緑（水色系）|
|在宅勤務|黄緑|
|休暇|グレー|
|退社|赤|
|直帰|赤|

※ 色覚多様性に配慮し、色だけでなく状態名も必ず表示する。

---

# 8. リアルタイム更新

画面更新には Server-Sent Events（SSE）を使用する。

状態変更が行われた場合

- 対象行のみ更新
- 約1秒間ハイライト表示
- 画面全体のリロードは行わない

# 9. スマートフォン画面

## レイアウト

```
────────────────────

自分

状態 ▼

行先

予定

[保存]

────────────────────

他社員一覧

────────────────────
```

### 特徴

- 自分の状態変更を最優先
- ボタンを大きく配置
- 片手操作を考慮したレイアウト
- 社員一覧は下部に表示

---

# 11. UI設計方針

- ホワイトボードと同じ感覚で利用できることを最優先とする。
- 状態変更は一覧画面から直接行う。
- 編集画面への遷移は最小限とする。
- 一覧画面を見るだけで社員全体の状況を把握できることを重視する。
- リアルタイム更新により、常に最新の情報を表示する。
