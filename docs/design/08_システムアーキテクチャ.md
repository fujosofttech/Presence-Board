# 08_システムアーキテクチャ

> Version: 1.0

---

# 1. アーキテクチャ概要

本システムは、オンプレミスのAIサーバー（ローカルLLM）と一般社員（PC/スマートフォン）が共通のREST APIを利用する、**API中心のWebアーキテクチャ**を採用する。

---

# 2. システム構成図

```mermaid
graph TD
    %% ユーザーからのアクセス
    User[一般社員 / PC・スマホ] -->|HTTPS| Nginx

    %% AIからのアクセス
    LocalAI[社内ローカルAIサーバー] -->|HTTPS (Bearer Token)| Nginx

    %% サーバー内部
    subgraph Oracle Cloud Always Free (Ubuntu)
        Nginx[Nginx] -->|Reverse Proxy| Gunicorn
        Gunicorn --> Django[Django Application]
        Django -->|ORM| PostgreSQL[(PostgreSQL)]
    end

    %% SSEのフロー
    Django -.->|SSE Stream| Nginx
    Nginx -.->|Server-Sent Events| User
```

---

# 3. コンポーネント詳細

## 3.1 Nginx (リバースプロキシ)
- HTTP(80)からHTTPS(443)へのリダイレクトを担当。
- Let's Encrypt (Certbot) による無料TLS証明書の自動更新。
- クライアントへの静的ファイル(`static/`)の直接配信。
- SSEの長期コネクションを維持するためのプロキシ設定（`proxy_buffering off`等）を適用。

## 3.2 Gunicorn (WSGI HTTP サーバー)
- Djangoアプリケーションを実行するためのPython WSGI HTTPサーバー。
- 複数ワーカープロセスを起動し、並行リクエストを処理する。

## 3.3 Django (Web フレームワーク / REST API)
- アプリケーションロジックの中核。
- **Django REST Framework (DRF)** を用いて、REST APIおよびOpenAPIドキュメントを提供する。
- HTMLテンプレートは極力利用せず、フロントエンド向け・AI向け共通のJSON APIとして機能させる。

## 3.4 PostgreSQL (データベース)
- リレーショナルデータベースとして全データを永続化。
- 本システムの主要なマスターデータ、およびステータス変更履歴を記録。

---

# 4. フロントエンドアーキテクチャ
- UIはブラウザ上で動作し、Bootstrap等を用いてレスポンシブ（PC・スマホ対応）に構築する。
- ページロード時にAPIから初期データを取得し、その後は **SSE (Server-Sent Events)** により他者の状態変更をリアルタイムに受信し、DOMを部分更新する。

---

# 5. AI First Designに基づく連携ポイント
- AIもブラウザUIを通さず、Nginx経由で直接DjangoのREST APIにアクセスする。
- 将来的に、Djangoアプリケーション内または並列のコンテナとして **MCPサーバー** を実装し、ローカルAIサーバーへ直接ツール（Functions）として機能を提供する拡張を見込む。
