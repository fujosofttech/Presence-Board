- 作成日: 2026-07-11
- 対象システム: Presence Board
- 対象読者: AIエージェント、設計者、開発者

---

# 1. 目的

本書は、AIがPresence Boardの実装・レビュー・テストを行う際の行動指針を定義する。

AIは開発チームの一員として行動するものとし、人間の設計意図を尊重し、品質・保守性・安全性を維持した実装を行うことを目的とする。

---

# 2. AIの役割

AIは以下の業務を担当する。

- ソースコードの実装
- リファクタリング
- Unit Testの作成
- コードレビュー
- ドキュメント作成補助
- バグ修正
- パフォーマンス改善提案

以下は担当しない。

- 要件決定
- 設計変更
- 業務仕様の決定
- セキュリティポリシーの変更
- データベース構造の変更（設計変更を伴うもの）

---

# 3. 基本原則

AIは以下を遵守する。

1. 要件を満たすこと
2. 設計書に従うこと
3. 推測で実装しないこと
4. 保守性を重視すること
5. テストを必ず作成すること
6. 不明点は人間へ確認すること

---

# 4. 実装前チェック

実装前に必ず確認する。

- 対応する要件
- 対応する設計書
- 対象タスク
- 対象ファイル
- 完了条件

不足している情報がある場合は実装しない。

---

# 5. 実装ルール

AIは以下を遵守する。

- 必要最小限の変更とする
- 既存コードの設計方針を尊重する
- 不要なリファクタリングを行わない
- コメントより読みやすいコードを書く
- 共通化は過剰に行わない

---

# 6. Python実装規約

- PEP8準拠
- 型ヒント必須
- docstringを記述する
- 定数は定数として管理する
- Magic Numberを使用しない
- Optionalを明示する

---

# 7. Django実装規約

- ORMを利用する
- Raw SQLは原則禁止
- Service層へ業務ロジックを配置する
- Serializerで入力検証を行う
- Transactionを適切に使用する
- N+1問題を避ける
- select_related、prefetch_relatedを適切に利用する

---

# 8. API実装規約

APIはRESTful設計に従う。

必ず以下を考慮する。

- HTTPステータスコード
- バリデーション
- 認証
- 権限
- エラーハンドリング
- ログ出力

---

# 9. フロントエンド実装規約

Vue実装時は以下を遵守する。

- Vue3
- TypeScript
- Composition API
- Pinia
- Vuetify
- コンポーネントは単一責任とする

---

# 10. テスト規約

実装時はUnit Testを作成する。

最低限確認する内容。

- 正常系
- 異常系
- 境界値
- 権限エラー
- バリデーション

---

# 11. レビュー規約

レビューでは以下を確認する。

- 要件を満たしているか
- 設計どおりか
- コーディング規約違反がないか
- セキュリティ上の問題がないか
- テストが十分か
- パフォーマンスに問題がないか

---

# 12. セキュリティ

AIは以下を必ず確認する。

- SQL Injection
- XSS
- CSRF
- 認証漏れ
- 権限漏れ
- 個人情報の取り扱い
- ログへの機密情報出力

---

# 13. ログ出力

ログは必要最小限とする。

以下は出力してはならない。

- パスワード
- トークン
- 個人情報
- 機密情報

---

# 14. エラー処理

例外は適切に処理する。

禁止事項。

- Exceptionの握りつぶし
- 原因不明のexcept
- エラー内容を隠すこと

---

# 15. AIが変更してよい範囲

AIは以下のみ変更できる。

```
backend/
frontend/
tests/
```

---

# 16. AIが変更してはいけない範囲

以下は人間の承認が必要。

```
docs/
requirements/
design/
decisions/
infrastructure/
```

設計変更が必要な場合は提案を行い、人間の承認後に変更する。

---

# 17. AIへの実装依頼

AIへ依頼する場合は以下を提示する。

- Task ID
- 対象設計書
- 対象ファイル
- 完了条件

例

```
Task: TASK-012

対象:
Employee API

対象ファイル:
backend/employees/views.py

完了条件:
・API実装
・Unit Test追加
・Ruffエラーなし
・mypyエラーなし
```

---

# 18. AI実装完了条件

AIは以下をすべて満たした場合のみ完了とする。

- 実装完了
- Build成功
- Ruff成功
- mypy成功
- Unit Test成功
- レビュー完了

---

# 19. AIが禁止される行為

AIは以下を行ってはならない。

- 推測で仕様を追加する
- 設計を勝手に変更する
- 要件にない機能を追加する
- ライブラリを無断で追加する
- データベース構造を変更する
- テストを省略する
- エラーを無視する

---
