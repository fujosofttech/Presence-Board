# 01_プロジェクト概要

> Version: 1.0

---

## 1. プロジェクト概要

### 1.1 プロジェクト名
社内行先在席管理システム
- 英語名称: Internal Presence Management System
- 略称: Presence Board

### 1.2 背景
現在、本社ではホワイトボードを利用して社員の出勤状況、在席状況および行先を管理している。
この運用では以下の課題がある。
- 客先常駐社員が利用できない
- 外出先から利用できない
- スマートフォンで利用できない
- 情報共有がリアルタイムではない
- 履歴を確認できない
- 将来的な機能追加が困難

これらを解決するため、本システムを開発する。

---

## 2. プロジェクト目的

本システムは次の目的を達成する。
- ホワイトボード運用を廃止する
- PCスマートフォンから状態を登録できるようにする
- 客先常駐社員も利用できるようにする
- 社員全員で在席状況をリアルタイム共有する
- DX推進の第一歩とする
- 将来の社内ポータル基盤を構築する

---

## 3. コンセプト：AI First Design

本プロジェクトは単なるWeb化にとどまらず、**「人と AI が共同利用する業務システム」**というコンセプトに進化している。
「AI が使えるようにする」のではなく、「AI を第一級の利用者（First-class Citizen）として設計する」ことを最大の特徴とする。

### 3.1 AI First Design 設計原則
- REST API を正式インターフェースとする
- OpenAPI を自動生成する
- API 仕様書と実装を一致させる
- エラーコードを機械可読にする
- 状態コードは言語非依存（OUT、REMOTE など）とする
- UI を経由せず API だけで全機能を利用できる
- 将来の MCP（Model Context Protocol）対応を考慮する

---

## 4. システム概要

本システムはクラウド上で稼働するWebアプリケーションである。
社員はブラウザのみで利用できる。専用アプリのインストールは不要である。
PCおよびスマートフォンから同一URLへアクセスする。

## 8. プロジェクト方針と長期ビジョン

本システムは単なる行先掲示板ではなく、将来の社内ポータルの基盤として設計する。
また、「2026 年に設計したが、2030 年でも古く感じないシステム」を目標とする。Version 1.0 で実装すべき基本機能と、将来の拡張（AI 連携など）を明確に分離して進行する。

将来的に以下の機能を追加できる設計とする。
- 社内電話帳
- お知らせ
- 会議室予約
- 備品管理
- ワークフロー
- 安否確認
