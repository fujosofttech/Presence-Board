# 03_業務フロー・ユースケース

> Version: 1.0

---

# 1. 文書の目的

本書は社内行先在席管理システムの業務フローおよびユースケースを定義する。

利用者がどのようなタイミングで本システムを利用し、どのような操作を行うかを明確にすることを目的とする。

本書を基に画面設計、DB設計、API設計を実施する。

---

# 2. 利用者・アクター区分

本システムの利用者は次の3種類とする。

|利用者 / アクター|概要|
|-------|------|
|本社勤務社員|本社を拠点として勤務する社員|
|客先常駐社員|客先企業で常駐勤務する社員|
|AIエージェント|自然言語を解釈し、API経由でシステム運用を代行するAI（ローカルLLM）|

---

# 3. 本社勤務社員の業務フロー

## 3.1 出社

社員は出社後、PCまたはスマートフォンからシステムへログインする。

現在状態を「在席」に変更し業務を開始する。

```
出社
    
ログイン
    
状態「在席」
    
業務開始
```

---

## 3.2 外出

外出時は一覧画面から状態を変更する。

入力項目

- 状態：外出
- 行先
- 予定期間（開始日時～終了日時）

保存後、全社員へリアルタイム反映する。

```
一覧画面
    
状態「外出」
    
行先入力
    
予定期間入力
    
保存
    
リアルタイム更新
```

---

## 3.3 帰社

帰社後は状態を「在席」に戻す。

```
一覧画面
    
状態「在席」
    
保存
```

---

## 3.4 会議

会議時は会議室または開催場所を行先として登録する。

予定期間には会議時間を入力する。

例

- 第1会議室
- 第3会議室
- Teams会議

```
状態「会議」

行先：第3会議室

予定期間
15:00～16:00
```

---

## 3.5 出張

出張では複数日に対応できるよう開始日時終了日時を登録する。

例

```
状態：出張

行先：大阪営業所

予定期間

2026/07/12 09:00

～

2026/07/13 18:00
```

---

## 3.6 在宅勤務

在宅勤務では状態のみ変更する。

追加情報の入力は不要とする。

```
一覧画面



状態「在宅勤務」



保存
```

---

## 3.7 退社

退社時は状態を「退社」に変更する。

```
一覧画面



状態「退社」



保存
```

---

## 3.8 直帰

外出後に会社へ戻らず帰宅する場合は「直帰」を選択する。

入力項目

- 状態：直帰
- 行先
- 予定期間（任意）

```
状態「直帰」



行先入力



保存
```

---

# 4. 客先常駐社員の業務フロー

## 4.1 通常勤務

客先常駐社員は通常時の状態を「客先常駐」とする。

勤務場所には所属する客先企業を表示する。

例

```
状態

客先常駐

勤務場所

株式会社
```

通常勤務中は状態変更を行わない。

---

## 4.2 休暇

休暇取得時は状態を「休暇」へ変更する。

```
客先常駐



休暇



保存
```

休暇終了後は再度「客先常駐」へ戻す。

---

# 5. AIエージェントの業務フロー（AI First Design）

本システムはAI First Designに基づき、AIが人間と同様にシステムを利用するフローを定義する。

## 5.1 自然言語による状態更新の代行
社員がチャットボット（AI）に対して「これからABC商事へ出かけます」と発言すると、AIが意図を抽出し、本システムのAPI（`PUT /api/v1/me/presence`等）を呼び出して状態を「外出」、行先を「ABC商事」に更新する。全社員に即時反映される。

## 5.2 社内状況の要約と回答
社員がAIに対して「今日、〇〇課で休暇を取っている人はいる？」と質問すると、AIがAPI（`GET /api/v1/presence`）を呼び出して最新の在席状況を取得・要約して回答する。

## 5.3 定期的なシステム運用代行
指定された時刻（例：朝9時）に、AIがAPI経由で本日の予定（事前登録データ等）を取得し、自動的に現在の状態へ反映させたり、未入力の社員に対して通知アクションを起こす。

---

# 6. リアルタイム更新

社員やAIエージェントが状態を変更した場合は全利用者へ即時反映する。

画面更新はServer-Sent Events (SSE) を利用する。

利用者によるリロード操作は不要とする。

---

# 7. 一覧画面の表示順

一覧画面では以下の優先順位で表示する。

1. ログイン中の利用者
2. 所属グループ順
3. 氏名順

所属課による表示切替を可能とする。

---

# 8. 状態一覧

本システムで利用する状態は以下とする。（※APIやDB内部ではAI First Designに基づき、言語非依存のコードを使用する）

|状態 (表示名)|内部コード|説明|
|------|------|------|
|在席|PRESENT|本社で勤務中|
|外出|OUT|外出中|
|会議|MEETING|会議中|
|出張|OUT|出張中|
|在宅勤務|REMOTE|在宅勤務中|
|客先常駐|CUSTOMER|客先企業で勤務中|
|休暇|HOLIDAY|休暇取得中|
|退社|LEAVE|退社済み|
|直帰|DIRECT_HOME|会社へ戻らず帰宅|

状態はマスタ管理とし、将来的な追加変更に対応できる設計とする。

---

# 9. 業務ルール

- 状態変更は一覧画面から直接実施する。
- 画面遷移を最小限とし、ホワイトボードと同等以上の操作性を目指す。
- 状態変更後は全社員へ即時反映する。
- 行先は自由入力とする。
- 将来的にお気に入り行先および最近使用した行先を候補表示できる設計とする。
